# 覚えたことまとめ

このメモは、GitHubやリポジトリの基本操作を整理したものです。
今後はこの形式でまとめていきましょう。

## GitHubとは

- 自分が書いたコードをWeb上に公開して、他の人が見たりダウンロードしたりできるサービスです。

## リポジトリとは

- GitHub上でプロジェクトを保存・管理する場所です。

## リポジトリの作成手順

1. GitHubのトップページから「New」を押す。
2. https://github.com/new に移動する。
3. リポジトリ名を入力する。
4. Descriptionに説明を入力する。
5. Visibilityで「Public」か「Private」を選ぶ。
6. 「Add a README」は基本的にオンにする。
7. 「Add .gitignore」は必要に応じて選ぶ。
   - `.gitignore`は、GitHubにアップロードしないファイルを指定する設定ファイルです。
8. 「Add a license」は必要なら設定する。
9. 内容を確認して「Create repository」を押す。

## リポジトリへアップロードする手順

1. 作業内容を保存する。
2. VS Codeのソース管理タブを開く。
3. 変更されたファイルを確認する。
4. 変更をステージして、コミットメッセージを入力する（例：`README.mdを更新`）。
5. コミットする。
6. ターミナルで次のコマンドを実行する。
   - `git push origin {ブランチ名}`
7. GitHubで反映されたことを確認する。

## コマンド備忘録

- `cd {ディレクトリ名}`
  - 指定したディレクトリに移動する。

- `git push origin {ブランチ名}`
  - 指定したブランチにローカルのコミットをアップロードする。

- `git checkout {ブランチ名}`
  - 今のブランチから指定したブランチに移動

- `git checkout -b {ブランチ名}`
  - 今のブランチから指定したブランチを作って移動

- `git pull origin {ブランチ名}`
  - 今いるブランチに指定したブランチの変更を取り込む

- `git stash`
  - 変更一時退避

- `git stash pop`
  - stashで変更していたものを元に戻す
